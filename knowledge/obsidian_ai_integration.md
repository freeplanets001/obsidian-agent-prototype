# Obsidian AI 連携完全ガイド (Copilot, Text Generator etc.)

Obsidian は完全なローカル・テキストベースのツールであるため、昨今の生成AI (GPT-4, Claude 3, Gemini) との親和性が極めて高いのが特徴です。AIプラグインを導入することで、ノートそのものをAIのコンテキスト（記憶）として活用する「第二の脳」の対話が可能になります。

---

## 1. 代表的なAI統合プラグイン

### 1.1 Obsidian Copilot
**概要:** 最も包括的で使いやすいAIアシスタントプラグインです。右サイドバーにチャット画面を表示し、ChatGPTのWeb画面のように対話できます。

**主な機能:**
- OpenAI, Anthropic, Google (Gemini), Groq, ローカルLLM (Ollama, LM Studio) に標準対応。
- **Context Awareness:** 「今開いているノートの内容」をワンクリックでコンテキストとしてAIに渡せる。
- **Vault QA:** Vault内の全ノートをベクトル化し、過去のメモ全体に基づいたRAG（検索拡張生成）チャットが可能。
- **コマンド実行:** エディタ上でテキストを選択し、右クリックから「要約」「翻訳」「文法チェック」などのカスタムプロンプトを実行、その場でテキストを置換・追記できる。

**設定方法のアドバイス:**
- 各サービスのAPIキー（例: Anthropic Console から取得した `sk-ant-xx...`）を取得して設定画面に貼り付けます。
- コストを抑える場合は、高速で安価なモデル（`claude-3-haiku-20240307` や `gpt-4o-mini`）をデフォルトに設定するのがおすすめです。

### 1.2 Text Generator
**概要:** プロンプトエンジニアリングとテキスト生成に特化した強力なプラグイン。

**特徴:**
- チャットインターフェースではなく、ノート内で `/tg` と打つことでテキストを次々と「生成（Autocomplete）」させる使い方がメイン。
- テンプレートにプロンプトを登録しておき、特定のノート（日報や会議録）の要約を一瞬で生成するフローの構築に優れている。

### 1.3 Smart Connections
**概要:** ノート同士の「意味的な繋がり（Semantic Similarity）」をAIが見つけ出します。

**特徴:**
- OpenAIのEmbeddings API（またはローカル手法）を使用し、全ノートの類似度を計算。
- 右サイドバーに「今見ているノートの内容と似ている過去のノート」を自動表示。
- Zettelkasten における「リンクの発見」をAIが自動でやってくれる画期的なツール。
- チャット機能 (Smart Chat) も内蔵しており、Vault全体へ質問が可能。

## 2. ローカルLLMの活用 (完全なプライバシー)

企業のリサーチメモや日記など、絶対にクラウド（OpenAIなど）へデータを送りたくない場合は、ローカルLLMを使用できます。

### 2.1 LM Studio または Ollama の導入
1. PC (Mac/Windows) に `Ollama` または `LM Studio` をインストール。
2. Llama 3 や Gemma、Mistral などのモデルをダウンロード。
3. アプリケーションでローカルAPIサーバー（例: `http://localhost:11434`）を立ち上げる。

### 2.2 プラグイン側の設定
- Copilot などのプラグインで、プロバイダーを `Ollama` または `LocalAI` に変更。
- Base URL を `http://127.0.0.1:11434` に指定。
- **メリット:** 完全にオフラインで機密情報を要約・翻訳可能。無料。
- **デメリット:** PCのスペック（特にGPUメモリ）に依存するため、最新のクラウドAIに比べると推論速度や精度で劣る。

## 3. 実践的なAIワークフロー

### 3.1 読書メモの自動構造化
1. KindleなどのハイライトをObsidianに取り込む。
2. Copilot プラグインを開き、ハイライトを選択。
3. **プロンプト:** `以下のハイライトから、著者の3つの主要な主張を抽出し、それぞれに見出しをつけてマークダウン形式で要約してください。`

### 3.2 アイデアの壁打ち (Brainstorming)
1. Canvas を開いてアイデアの断片を配置する。
2. 煮詰まったら右サイドバーの Copilot を開き、`このノートの文脈を考慮して、次に検討すべきリスクを3つ挙げて` と送信。
3. 返ってきた回答をそのままCanvasにカードとしてドラッグ＆ドロップ。

### 3.3 タスクの自動分解
1. デイリーノートに大雑把な長文（例: `今週中にWebサイトのデザインをリニューアルして、金曜の実装に間に合わせる`）を書く。
2. 長文を選択し、AIカスタムコマンド「タスク化」を実行。
3. **システムプロンプト設定例:** `入力されたテキストを分析し、具体的なアクションアイテムのチェックリスト（- [ ] ）に分解してください。`
4. 即座にエディタ上でタスクリストが展開される。

## 4. AIとObsidianの未来

Obsidian は単なるエディタから「AIの記憶ツール」になりつつあります。Vaultに蓄積された数年分のノートは、一般的なAIモデルが持ち得ない「あなた独自のコンテキスト」です。このデータとAIを同期・連携させることで、真の意味で「もう一つの脳」を体現することが可能です。
