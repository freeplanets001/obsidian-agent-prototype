# Obsidian ワークフローとノート術 (Zettelkasten / PARA / MOC)

Obsidianはその柔軟性ゆえに「どう使えばいいかわからない」と迷うユーザーが多くいます。ここでは世界中で実践されている代表的な3つのナレッジマネジメント・フレームワークとそのObsidianでの実装方法を解説します。

---

## 1. Zettelkasten (ツェッテルカステン)

ドイツの社会学者ニクラス・ルーマンが考案した「情報カード」システムをデジタルに適応させた手法です。「フォルダを使わず、リンク（繋がり）で知識を構築する」ことが最大の特徴です。

### 1.1 Zettelkasten の3種類のノート
1. **Fleeting Notes (走り書きノート):** アイデアが浮かんだ瞬間に書く一時的なメモ。デイリーノートに書くことが多い。
2. **Literature Notes (文献ノート):** 本や記事を読んで、自分の言葉で要約したノート。
3. **Permanent Notes (永久ノート):** 上記のメモから、一つの概念・アイデアを完全に独立した文章として抽出したノート。「1ノート＝1アイデア (Atomicity)」の原則に従う。

### 1.2 Obsidianでの実践方法
- **デイリーノート機能**を使ってFleeting Notesを記録する。
- 永久ノートには「固有のID（タイムスタンプなど）」をファイル名につけ、必ず他のノートへ `[[リンク]]` を張る。
- フォルダ階層を作らず、1つの `Zettelkasten` フォルダに全ノートをフラットに放り込む。
- **グラフビュー** を使って、アイデア同士の予期せぬ繋がりを発見する。

## 2. PARA メソッド

Tiago Forte 氏が提唱する「情報をアクション可能性（Actionability）に基づいて整理する」手法。デジタルデータのフォルダ構成として非常に人気です。

### 2.1 PARAの4つのカテゴリ
1. **P - Projects (プロジェクト):** 明確な完了目標と期限があるもの。（例: 「2025年冬の旅行計画」「Obsidianエージェント開発」）
2. **A - Areas (エリア/責任領域):** 継続的に維持・管理する必要があるが、明確な完了がないもの。（例: 「健康」「財務」「人事管理」）
3. **R - Resources (リソース):** 個人的な興味、将来役立つかもしれないリファレンス情報。（例: 「読書ノート」「プログラミングの学習メモ」「レシピ」）
4. **A - Archives (アーカイブ):** 完了したプロジェクトや、現在は非アクティブなエリア・リソース。

### 2.2 Obsidianでの実践方法
- Vault (ルートフォルダ) に `1-Projects`, `2-Areas`, `3-Resources`, `4-Archives` という4つのフォルダを作る。
- **Projectsの管理:** デイリーノートから、作業中の Projects フォルダ内ファイルへリンクを貼る。
- **Archivesの活用:** 不要になったら捨てるのではなく、4-Archivesフォルダに移動させることで、検索（Ctrl+F / Cmd+F）の対象としては残したまま、視界をクリアに保つ。

## 3. MOC (Map of Content / コンテンツの地図)

Nick Milo 氏が提唱。フォルダによる厳格な分類（パラ）と、Zettelkastenの無秩序なリンク状態の「中間」を狙うアプローチです。

### 3.1 MOCの概念
ノートが数十、数百と増えてくると、リンクだけでは迷子になります。そこで、特定のテーマの「目次（ハブとなるノート）」を作成し、そこから関連ノートへリンクを貼ります。これが MOC です。

### 3.2 Obsidianでの実践方法
- `+ MOC 宇宙` や `[[哲学 MOC]]` といったタイトルのノートを作成。
- そのノートの中に、宇宙に関連する全てのノートへのリンクをリスト化する。
- ホーム画面となる「Home MOC」または「Index」ノートを作り、そこから各「Sub MOC」へリンクを張る（例：`[[テクノロジー MOC]]` `[[読書記録 MOC]]`）。
- **Dataview プラグイン** との相性が抜群。手動でリンクを書かなくても、タグ `#cosmos` が付いたノート一覧をMOCノートに自動表示できる。

## 4. デイリーノート中心のワークフロー (Daily Note Centric)

Obsidian を使う上で最も推奨される基本的な手法。

### 4.1 メリット
ファイル名をどうするか、どこに保存するか悩む「摩擦」をゼロにします。「とりあえず今日のノートを開いて全てをそこに書く」というルールにするため、記録のハードルが極端に下がります。

### 4.2 具体的なフロー
1. 朝、Obsidianを開くと自動で本日のノート（例: `2025-01-15.md`）が開く。
2. その日のタスク（Taskプラグインを使用）、会議録、思いついたアイデア、日記をすべて時系列で箇条書き（`- `）にする。
3. もし特定のテーマについてのメモが長くなりそうだと感じたら、その場で `[[新しい概念]]` とブラケットで囲み、リンク先ファイルを作成してそちらに詳細を書く。
4. 元のデイリーノートには `[[新しい概念]] についてのリサーチを開始` と履歴だけが残る。

## 5. フレームワークの統合 (ハイブリッド型)

どれか1つに絞る必要はありません。多くの熟練ユーザーは以下のように組み合わせています。

- **ルートフォルダ:** PARA メソッド (`Projects`, `Areas`, `Resources`, `Archives`)
- **Projects フォルダ内:** プロジェクト専用のダッシュボードノート（MOC）を置く。
- **情報の投入:** 全ての起点はデイリーノート（Daily Note Centric）。
- **Resources フォルダ内:** Zettelkasten のようにリンク主体でメモを育てる。

この柔軟な「自分だけのハイブリッドシステム」を構築できることこそが、Obsidian が他のどのツール（Notion、Evernoteなど）よりも優れている理由です。
