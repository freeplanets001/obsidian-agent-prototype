# Obsidian 同期（Sync）とバックアップ 完全ガイド

Obsidian はデータがすべてローカルの Markdown ファイル（Vault）として保存されるため、同期とバックアップの方法もユーザー自身が決定する必要があります。各手法のメリット・デメリットを理解し、最適な環境を構築しましょう。

---

## 1. 公式サービス: Obsidian Sync

Obsidian 開発チームが提供する公式の同期サービスです（有料プラン）。

**メリット:**
- **最も確実で安全:** エクスペリエンスがシームレス。設定、プラグイン、テーマまで全て完璧に同期。
- **エンドツーエンド暗号化 (E2EE):** パスワードで完全に暗号化され、サーバー側でもデータは読めません。
- **バージョン履歴:** 1年間の変更履歴（バージョンコントロール）が保存され、いつでも復元可能。
- **全てのプラットフォーム対応:** iOS, Android, Windows, Mac, Linux 間で最も安定。

**デメリット:**
- 有料（月額約4〜8ドル程度、プランによる）。

**こんな人におすすめ:**
- お金を払ってでも「設定不要の安心・安全・手間なし」を選びたい人。
- 会社用と個人用など、機密性の高いデータを複数端末で扱う人。

## 2. クラウドストレージ (iCloud Drive / Google Drive / Dropbox)

OSが提供する仕組みや汎用のクラウドストレージファイルアプリを利用する方法です。

### 2.1 iCloud Drive (Apple エコシステム向け)
MacとiPhone/iPadを使っているユーザーにとっての定番。

**メリット:**
- Apple端末間なら無料で即座に同期される。
- iOSアプリでも「iCloud用Vault」として直接開ける。

**デメリット:**
- Windows/Android端末との同期には不向き。
- 「最適化」機能（ファイルをクラウドのみに置く機能）がオンになっていると、インデックス作成時にフリーズしたり、ファイルが見つからないエラーが起きる。**必ず「ファイルのダウンロードを維持」に設定**が必要。

### 2.2 Google Drive / Dropbox
PC間での同期に便利です。

**メリット:**
- すでに使っているクラウドの容量を使用できる。

**デメリット:**
- iOS/Android の Obsidian アプリは、標準ではGoogle DriveやDropbox内のフォルダを直接Vaultとして開けない。（サードパーティアプリを活用する迂回手段が必要）
- デスクトップ版の「オンデマンド同期」機能をオフにし、常にローカルに保存する設定にする必要がある。

## 3. GitHub (Git プラグイン) による同期とバックアップ

プログラマーやGitに慣れている人向けの設定です。 Obsidian Git プラグインを使います。

**メリット:**
- 無料で高度なプロ用バージョン管理が利用できる。
- 全ての変更履歴が残る（コミットログ）。
- パブリックリポジトリを使えば、Obsidian Publishの代わりのような使い方も（無料ブログ化）。

**デメリット:**
- Gitの知識が必要（コンフリクト発生時の解決など）。
- モバイル環境（iOS/Android）での動作設定がやや煩雑（Working CopyやTermuxなどとの連携が必要な場合がある）。
- 大きなファイル（動画、重いPDF）を大量にコミットするとGitリポジトリが肥大化しやすい。

---

## 4. Syncthing (P2P同期)

クラウドサーバーを介さず、端末同士を直接P2Pで繋ぐオープンソースの同期ツール。

**メリット:**
- 完全無料かつ容量無制限。
- サードパーティのサーバーにデータを預けないため、究極のプライバシー。

**デメリット:**
- 同期したいデバイスが双方とも起動してネットワークに繋がっている必要がある。
- iOS版クライアントが存在しない（または公式ではなく高価）ため、iPhoneユーザーには致命的。Android/PC間なら非常に優秀。

---

## 5. バックアップ戦略 (3-2-1 ルール)

同期（Sync）はバックアップ（Backup）ではありません。間違って上書きした場合、同期されると他の端末でも上書きされてしまいます。

**推奨されるバックアップ戦略:**

1. **Obsidian Git**: 日々のスナップショット（コミット）を自動で作成。
2. **Local Backup プラグイン**: Vault全体のZIPファイルを定期的に別ドライブに書き出す。
3. **クラウドバックアップ**: PC自体のバックアップ（Backblaze, Time Machine, OneDrive等）にVaultを含める。

**コミュニティプラグインの設定例:**
- プラグイン: `Obsidian Git`
- 設定:
  - `Vault backup interval`: 30分（30分ごとに自動コミット）
  - `Auto backup after file change`: オン
  - `Commit message`: `Vault backup: {{date}}`

---

## 6. プラグインや設定 (.obsidian) の同期についての注意

Vault フォルダ内には `.obsidian` という隠しフォルダがあり、ここにテーマや設定、インストールした全プラグインが格納されています。

- 複数デバイスで全く同じ環境を使いたい場合は、この `.obsidian` フォルダも同期対象にします。
- PCではダークテーマ、スマホではライトテーマといったように「端末ごとに設定を変えたい」場合は、Obsidian Sync の設定で「テーマの同期をオフ」にするか、手動同期環境であれば別々の `.obsidian` プロファイル（例: `.obsidian-mobile`）を使用する機能（Advanced URIプラグイン等と連携）を検討します。

---

## 7. 結論・推奨

1. **予算がある・手間をかけたくない・iOSとWindowsを使う:** `Obsidian Sync` 一択。
2. **Appleデバイス（Mac + iPad/iPhone）しか使わない:** `iCloud Drive`。（最適化オフ必須）
3. **IT知識があり、無料で高度な管理がしたい:** `Obsidian Git`。
4. **Android と Windows しか使わない:** `Syncthing` または専用フォルダ同期アプリ。
